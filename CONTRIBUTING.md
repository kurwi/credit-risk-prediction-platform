# Contributing Guidelines

Thank you for your interest in contributing to the Credit Risk Prediction Platform!

## How to Contribute

- Fork the repository and create your branch from `main`.
- Write clear, well-documented code and add/modify tests as needed.
- Ensure all tests pass (`pytest -v --cov=src`).
- Follow PEP 8 and project code style.
- Submit a pull request with a clear description of your changes.

## Code of Conduct

Please review the CODE_OF_CONDUCT.md before contributing.

## Reporting Issues

- Use the GitHub Issues tab to report bugs or request features.
- Provide detailed steps to reproduce bugs.

## Development Environment

- Python 3.10+
- Install dev dependencies: `pip install -r config/requirements-dev.txt`

## Contact

For questions, open an issue or contact the maintainer listed in the README.
